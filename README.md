📊 充足率・流動可視化マップ（JapanMapProject）

概要
本プロジェクトは、日本全国の都道府県データを対象に、
充足率（アポ数 / キャパ数） および 地域間の流動（フロー） を
地図上で直感的に可視化する Web アプリケーションです。
月・日付単位でのデータ切替や、特定地域を起点としたフロー表示など、
業務データの分析・可視化を想定したUI設計を意識して実装しています。

🎯 想定ユースケース
営業・人材・拠点配置などのKPI可視化
地域ごとの充足状況の把握
拠点間・地域間の流動分析
DX / BI / データ分析案件のプロトタイプ
※ 使用している数値データはデモ用に生成したものです。

🗺 主な機能
1. 都道府県別「充足率」可視化
各都道府県に 円グラフ型マーカー を表示
充足率を視覚的に把握可能
中央にパーセンテージを表示

2. 月・日付切替
月（8月 / 9月）ごとにデータ切替
日付セレクトにより 日次変化を即座に反映

3. 地域間フロー（矢印）表示
主要都道府県をクリックすると、
当該地域を起点とする 流動データを矢印で表示
矢印の太さ・ラベルで数量を表現
ズーム操作に追従して矢印を再描画

4. インタラクション設計
マウスオーバーによる強調表示
クリックによる表示切替
ツールチップによる詳細情報表示

🧩 技術構成
フロントエンド
HTML / CSS / JavaScript（Vanilla JS）
Leaflet.js（地図描画ライブラリ）
Canvas API（円グラフアイコン生成）
データ管理
CSV → Python により JS データ生成
月別データファイル切替（data.js, data_9.js 等）
Flow データも同様に分離管理
ローカル開発環境
Python 標準ライブラリによる簡易 HTTP サーバー
キャッシュ無効化対応（開発効率向上）

📂 ファイル構成（抜粋）
├─ index.html        # UI・画面構成
├─ map.js            # 地図描画・インタラクション制御
├─ coords.js         # 都道府県座標データ
├─ data.js           # 8月データ
├─ data_9.js         # 9月データ
├─ flow_data.js      # 8月フローデータ
├─ flow_data_9.js    # 9月フローデータ
├─ server.py         # ローカル開発用HTTPサーバー

🚀 起動方法（ローカル）
python server.py
起動後、ブラウザで以下にアクセス：
http://localhost:8000
（同一LAN内の場合）表示される IP アドレス

💡 工夫したポイント
地図 × KPI × フロー を一画面で理解できる構成
データ構造を月単位で分離し、拡張しやすい設計
Canvas を用いた 動的アイコン生成
実務を想定した「クリック → 深掘り」導線
